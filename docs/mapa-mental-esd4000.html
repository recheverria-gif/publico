Mapa mental: alta de relación de grupo familiar para cónyuge

Resumen visual basado en el fuente RPGLE ESD4000. Esta página explica los campos mínimos obligatorios, los valores identificados en el código, el propósito general del programa y los archivos principales usados, además de las validaciones que aplica al crear una relación de tipo Grupo Familiar para un cónyuge.

Mapa mental

  • ESD4000 → Nueva relación grupo familiar / cónyuge

    El programa administra relaciones entre un cliente/cuenta principal y una persona o entidad relacionada.

    Tipo relación: F Caso buscado: Cónyuge Código exacto de cónyuge no visible en este fuente

    • 0. Propósito general

      ESD4000 es un programa RPGLE de mantenimiento de relaciones entre clientes/cuentas y terceros relacionados. Permite consultar, crear, validar, listar, actualizar y marcar para eliminación registros de relación en CUREL y WKCUREL.

      • Administra relaciones como grupo familiar, beneficiarios, firmantes, referencias y representantes.
      • Valida reglas según el tipo de relación.
      • Actualiza archivos de trabajo para procesos de aprobación.
      • Envía respuesta al invocador con datos y errores.

    • 0.1 Archivos principales usados

      El programa trabaja con archivos maestros, archivos de trabajo, catálogos y archivos de control.

      • CUMST: maestro de clientes
      • CUENT: maestro de entidades/no clientes
      • CUREL: maestro de relaciones
      • WKCUMST: work de clientes
      • WKCUENT: work de entidades/no clientes
      • WKCUREL: work de relaciones
      • CNOFC: tabla de códigos y descripciones
      • CUFIN, CUPEP: datos complementarios del cliente
      • CNTRLCNT, CNTRLBTH, CNTRLBRN: control/configuración
      • ACMSTL0, DEALSL0, LCMSTL0, DCMSTL0, ROCOLL0: productos/cuentas relacionadas

    • 1. Contexto funcional

      • e01rac: identifica al titular principal.
      • e01rnu: identifica a la persona relacionada.
      • e01rtp = 'F': indica grupo familiar.
      • e01uc1: parentesco del relacionado respecto al titular.

    • 2. Flujo mínimo

      • Primero puede usarse h01ope = '0001' para preparar un nuevo registro.
      • La grabación real ocurre en otra operación que cae en valrecord.
      • En valrecord se valida la existencia del relacionado y luego la relación tipo F.
      • Si no hay errores, el programa graba en WKCUREL.

    • 3. Campos mínimos obligatorios

      • h01ope: operación de grabación/validación.
      • e01rcd: tipo del registro principal.
      • e01rac: número del cliente/cuenta principal.
      • e01rtp = 'F': grupo familiar.
      • e01rcn: tipo de relacionado.
      • e01rnu: número del relacionado.
      • e01uc1: código de parentesco.

    • 4. Valor específico para cónyuge

      • El programa valida e01uc1 en la tabla de códigos CNOFC.
      • La clase usada es 9W = parentesco.
      • Por tanto, e01uc1 debe contener el código de parentesco equivalente a cónyuge.
      • Ese código exacto no aparece hardcodeado en este fuente.

    • 5. Validación específica del tipo F

      • valrecord detecta e01rtp = 'F'.
      • Llama a vfamilyrefer.
      • vfamilyrefer valida que e01uc1 exista en cnocfl = '9W'.

    • 6. Validación implícita del relacionado

      • Si e01rcn = 'C', el relacionado debe existir en CUMST o WKCUMST.
      • Si no es cliente, debe existir en CUENT o WKCUENT.
      • Si no existe, el alta falla con error.

    • 7. Campos no obligatorios en este caso

      • No se ve obligatoriedad para e01per (porcentaje).
      • No se ve obligatoriedad para e01amt (monto).
      • No se ve obligatoriedad para fechas e01idm/e01idd/e01idy.
      • No se ve obligatoriedad para e01rmk o e01dsc.

Resumen práctico

Propósito general

ESD4000 mantiene relaciones entre clientes o cuentas principales y personas o entidades relacionadas. Soporta consulta, alta, validación, listado, actualización y marcado para eliminación de relaciones, usando archivos maestros y archivos de trabajo para aprobación posterior.

Archivos principales usados

Maestros: CUMST, CUENT, CUREL

Work: WKCUMST, WKCUENT, WKCUREL

Catálogos: CNOFC

Complementarios: CUFIN, CUPEP

Control: CNTRLCNT, CNTRLBTH, CNTRLBRN

Campos mínimos

h01ope
Operación de grabación/validación. Nota: '0001' prepara, pero no graba.
e01rcd
Tipo del registro principal.
e01rac
Número del cliente o cuenta principal.
e01rtp
'F' = Grupo Familiar.
e01rcn
Tipo de relacionado. Si el cónyuge es cliente, normalmente 'C'.
e01rnu
Número del relacionado (cónyuge).
e01uc1
Código de parentesco. Debe corresponder a cónyuge en la tabla 9W.

Valores identificados en el fuente

e01rtp = 'F'

e01rcn = 'C' si el cónyuge está registrado como cliente.

Tabla de parentesco = '9W'

Valor exacto del código de cónyuge: no visible en ESD4000

Ejemplo conceptual

h01ope = [operación de grabación]
e01rcd = [tipo principal]
e01rac = 123456789
e01rtp = 'F'
e01rcn = 'C'
e01rnu = 987654321
e01uc1 = [código de parentesco para cónyuge en 9W]

Conclusión

Para dar de alta una relación de grupo familiar para cónyuge, el programa exige principalmente la identificación del titular, la identificación del relacionado, el tipo de relación F y el parentesco en e01uc1. El dato faltante a confirmar fuera de este fuente es el código exacto de cónyuge dentro de la tabla 9W.
