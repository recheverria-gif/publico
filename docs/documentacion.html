Documentación técnica de QTREE

Esta página resume la responsabilidad de cada fuente del sistema QTREE y explica cómo se relacionan para construir el árbol de llamados de un programa IBM i. El objetivo es ofrecer una vista clara del proceso desde el comando inicial hasta la generación del archivo TREETXT.

Flujo general del sistema

El sistema parte con un comando, prepara archivos temporales, obtiene referencias mediante DSPPGMREF, filtra los resultados con SQL y deja la generación visual del árbol a un programa RPG.

ARBOL
→
ARBOL01CL
→
ARBOL02CL
→
DSPPGMREF + SQL
→
ARBOLRPG
→
TREETXT

Explicación por componente

CMD

QTREE.ARBOL

Define el comando de entrada del sistema. Recibe el programa principal a analizar y opciones como inclusión de archivos, programas de servicio y filtros relacionados con actualización o salida.

CL

QTREE.ARBOL00CL

Programa auxiliar de instalación. Compila los CL principales y crea el comando ARBOL para dejar la herramienta disponible para ejecución.

CL

QTREE.ARBOL01CL

Actúa como controlador principal. Limpia QTEMP, crea o prepara objetos temporales, invoca a ARBOL02CL y finalmente muestra el resultado almacenado en TREETXT.

CL

QTREE.ARBOL02CL

Es el motor de análisis. Ejecuta DSPPGMREF sobre el programa objetivo, separa programas y archivos, filtra resultados con SQL, realiza overrides y llama al RPG que construye la salida visual.

RPG

QTREE.ARBOLRPG

Construye el árbol visual. Formatea líneas, dibuja conectores, lista archivos utilizados y profundiza la exploración llamando de nuevo a ARBOL02CL cuando debe bajar a programas hijos.

RPG

QTREE.ARBOLRPGX

Variante de ARBOLRPG. Mantiene el mismo propósito general pero aparenta tener diferencias menores de formato, textos o comportamiento auxiliar.

SQL

F_PGM / F_PGM_GRP / F_PGM_SRV / FILTROFIL

Son sentencias SQL auxiliares que copian o filtran referencias temporales para separar programas, service programs y archivos antes de la generación del árbol.

DDS

QTREE.TREEFILE

DDS del archivo físico TREEFILE. Sirve como apoyo al control de archivos usados y su tipo de utilización dentro del análisis.

Resumen tabular

Objeto Tipo Rol principal
QTREE.ARBOL CMD Interfaz de entrada para lanzar el análisis.
QTREE.ARBOL00CL CL Compila componentes y crea el comando.
QTREE.ARBOL01CL CL Prepara temporales y muestra la salida final.
QTREE.ARBOL02CL CL Extrae referencias y organiza datos para el RPG.
QTREE.ARBOLRPG RPG Construye visualmente el árbol de llamados.
QTREE.ARBOLRPGX RPG Variante alternativa del generador visual.
QTREE.F_PGM SQL Filtra solo objetos tipo *PGM.
QTREE.F_PGM_GRP SQL Agrupa referencias de programas base.
QTREE.F_PGM_SRV SQL Incluye *PGM y *SRVPGM.
QTREE.FILTROFIL SQL Prepara referencias de archivos.
QTREE.TREEFILE DDS Define archivo auxiliar para control de uso de archivos.

Conclusión

QTREE sigue una arquitectura clara: un comando de entrada, CL de control y preparación, SQL auxiliar para separar la información y RPG para producir la representación final del árbol. Esta separación permite que el análisis técnico y la presentación visual estén desacoplados, facilitando el mantenimiento de la herramienta.
